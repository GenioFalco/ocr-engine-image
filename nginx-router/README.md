# Глобальный Nginx Роутер (Оркестратор Сервера)

Этот проект служит единой точкой входа (Gateway) для всех доменов на этом сервере.
Он слушает **порт 80** и распределяет запросы по разным внутренним docker-контейнерам или проектам на сервере.

## Как добавить новый проект (домен) на сервер?

1. Создайте ваш проект в любой папке, запустите его в Docker на любом свободном порту (например, 8000, 3000, 5173).
   *Убедитесь, что ваш проект пробрасывает порты на локальную машину (например, `ports: "3000:3000"`).*
2. В папке `conf.d` этого роутера создайте файл с названием вашего домена, например `my-site.com.conf`.
3. Заполните его вот по такому шаблону:

```nginx
server {
    listen 80;
    server_name my-site.com www.my-site.com;

    location / {
        # Замените 3000 на порт вашего запущенного проекта
        proxy_pass http://host.docker.internal:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

4. Перезагрузите оркестратор, чтобы изменения вступили в силу:
   ```bash
   docker-compose restart orchestrator
   ```

Всё! Теперь ваш новый домен автоматически направляет трафик на нужный порт. SSL сертификатами занимается балансировщик перед этим сервером.
